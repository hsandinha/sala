Introdução à Modelagem do Sistema de Coworking

Bem-vindo ao visualizador interativo da modelagem do sistema de agendamento de coworking e salas de reunião. Este documento digital apresenta a arquitetura e os componentes chave planejados para a aplicação, conforme detalhado no relatório original. O sistema é projetado para atender dois perfis principais: Administradores de espaços (responsáveis pela gestão das salas) e Usuários Finais (clientes que desejam alugar os espaços).

A tecnologia base para este sistema inclui Node.js para o backend, React Native para o aplicativo móvel e MongoDB como banco de dados. Navegue pelas seções utilizando o menu à esquerda para explorar o modelo de dados, o design das telas para cada perfil de usuário, a estrutura da API backend e outras considerações importantes do projeto.

O objetivo desta visualização é fornecer uma compreensão clara e acessível de como o sistema será estruturado e como suas diferentes partes interagem.

Modelo de Dados: Coleção `users`

Esta coleção armazena informações de todos os usuários do sistema, sejam eles administradores dos espaços ou usuários finais que realizam as reservas. O campo `role` distingue os dois tipos de usuários.

Campos da Coleção `users`:

{ "_id": "ObjectId", // Chave primária "name": "String", // Nome completo "email": "String", // Email (único, para login) "password": "String", // Senha (armazenar hash) "role": "String", // "admin" ou "user" "phone": "String", // (Opcional) Telefone de contato "cpf_cnpj": "String", // (Opcional) Para identificação e faturamento "createdAt": "Date", "updatedAt": "Date" }

Modelo de Dados: Coleção `spaces`

A coleção `spaces` contém informações sobre os edifícios ou locais físicos de coworking que abrigam as diversas salas. Cada espaço é gerenciado por um administrador específico, referenciado pelo campo `adminId`.

Campos da Coleção `spaces`:

{ "_id": "ObjectId", "name": "String", // Ex: "Coworking Central SP", "Unidade Savassi BH" "address": { "street": "String", "number": "String", "complement": "String", "neighborhood": "String", "city": "String", "state": "String", "zipCode": "String", "coordinates": { // Opcional, para mapas "latitude": "Number", "longitude": "Number" } }, "adminId": "ObjectId", // Referência ao usuário (admin) proprietário/gestor do espaço "amenities": ["String"], // Ex: ["wifi", "cafeteria", "estacionamento"] "operatingHours": { // Horários de funcionamento do espaço "monday": { "open": "HH:MM", "close": "HH:MM", "isClosed": "Boolean" }, "tuesday": { "open": "HH:MM", "close": "HH:MM", "isClosed": "Boolean" } // ... demais dias }, "photos": ["String"], // URLs de fotos do espaço "createdAt": "Date", "updatedAt": "Date" }

Modelo de Dados: Coleção `rooms`

Esta coleção detalha cada sala de reunião individual ou posto de trabalho disponível para aluguel dentro de um `space`. Inclui informações sobre capacidade, tipo, comodidades, preços e períodos de manutenção.

Campos da Coleção `rooms`:

{ "_id": "ObjectId", "spaceId": "ObjectId", // Referência à coleção 'spaces' "name": "String", // Ex: "Sala de Reunião Alfa", "Estação de Trabalho 05" "type": "String", // Ex: "meeting_room", "coworking_desk", "private_office" "capacity": "Number", // Número de pessoas que a sala comporta "description": "String", // Descrição detalhada da sala "amenities": ["String"], // Ex: ["projetor", "quadro_branco", "ar_condicionado"] "photos": ["String"], // URLs de fotos da sala "pricing": { "perHour": "Number", "perDay": "Number", // (Opcional) "perMonth": "Number" // (Opcional, para escritórios privativos) }, "isAvailable": "Boolean", // Status geral (pode ser dinâmico ou um default) "maintenancePeriods": [ // Períodos em que a sala está indisponível para manutenção { "startDateTime": "Date", "endDateTime": "Date", "reason": "String" } ], "createdAt": "Date", "updatedAt": "Date" }

Modelo de Dados: Coleção `bookings`

A coleção `bookings` é crucial, pois armazena todas as informações sobre as reservas realizadas pelos usuários. Ela vincula um usuário (`userId`) a uma sala específica (`roomId`) por um determinado período, incluindo detalhes de pagamento e o status da reserva.

Campos da Coleção `bookings`:

{ "_id": "ObjectId", "userId": "ObjectId", // Referência à coleção 'users' (quem reservou) "roomId": "ObjectId", // Referência à coleção 'rooms' (qual sala) "spaceId": "ObjectId", // Referência à coleção 'spaces' (em qual espaço, para facilitar consultas) "startDateTime": "Date", // Data e hora de início da reserva "endDateTime": "Date", // Data e hora de fim da reserva "totalPrice": "Number", // Preço total calculado "status": "String", // Ex: "pending_payment", "confirmed", "cancelled_by_user", "cancelled_by_admin", "completed", "no_show" "protocolNumber": "String", // Número único de protocolo da reserva "paymentDetails": { "method": "String", // Ex: "credit_card", "pix", "bank_transfer", "cash_on_site" "transactionId": "String", // (Opcional) ID da transação do gateway de pagamento "status": "String", // Ex: "paid", "pending", "failed" "paidAt": "Date" // Data do pagamento }, "notes": "String", // (Opcional) Observações do usuário ou admin "createdAt": "Date", "updatedAt": "Date" }

Telas da Aplicação: Perfil Administrador

Esta seção descreve as telas planejadas para o perfil de Administrador. O administrador é responsável pela gestão completa dos espaços, salas, reservas e pagamentos, garantindo o bom funcionamento e a rentabilidade do coworking.

Autenticação

  • LoginAdminScreen: Campos para email e senha. Link para "Esqueci minha senha".
  • ForgotPasswordAdminScreen: Campo para email para envio de link de redefinição.
  • ResetPasswordAdminScreen: Campos para nova senha e confirmação.

Dashboard/Home (Admin)

  • Visão geral: Número de reservas para hoje/semana, salas mais ocupadas, receita recente.
  • Atalhos para gerenciamento de salas, reservas e pagamentos.
  • Notificações (novas reservas, pedidos de cancelamento).

Gerenciamento de Espaços

  • MySpacesScreen: Lista os espaços gerenciados pelo admin. Opção para adicionar novo espaço.
  • SpaceFormScreen: Formulário para criar/editar dados do espaço (nome, endereço, comodidades, horários, fotos).

Gerenciamento de Salas

  • RoomListScreen: Lista de salas (filtrável por espaço). Opção para adicionar nova sala. Exibe nome, tipo, capacidade, status.
  • RoomFormScreen: Formulário para criar/editar dados da sala (nome, tipo, capacidade, descrição, comodidades, fotos, preços).
  • RoomAvailabilityScreen: Visualização em calendário da disponibilidade. Opção para bloquear/desbloquear horários (manutenção). Adicionar/editar/remover períodos de manutenção.

Gerenciamento de Reservas (Admin)

  • BookingListAdminScreen: Lista de todas as reservas com filtros (data, sala, status, usuário). Exibe protocolo, sala, usuário, data/hora, status.
  • BookingDetailAdminScreen: Detalhes completos da reserva. Opções para confirmar reserva, registrar pagamento, cancelar reserva.

Gerenciamento de Pagamentos (Admin)

  • PaymentListAdminScreen: Lista de transações/pagamentos com filtros. Visualizar status (pago, pendente, falhou). Pode ser integrado à `BookingDetailAdminScreen`.

Configurações (Admin)

  • AdminProfileScreen: Editar dados do perfil do administrador.
  • NotificationSettingsAdminScreen: Configurar preferências de notificação.
  • OperatingHoursSpaceScreen: Gerenciar horários de funcionamento padrão do(s) espaço(s).

Telas da Aplicação: Perfil Usuário Final

Aqui são detalhadas as telas para o Usuário Final – o cliente que busca alugar salas de coworking ou reunião. O foco é na facilidade de busca, reserva e gerenciamento das suas próprias locações.

Autenticação

  • LoginUserScreen: Campos para email e senha. Links para "Esqueci minha senha" e "Cadastre-se".
  • SignUpUserScreen: Formulário de cadastro (nome, email, senha, etc.).
  • ForgotPasswordUserScreen: Campo para email para envio de link de redefinição.
  • ResetPasswordUserScreen: Campos para nova senha e confirmação.

Home/Explorar Salas

  • RoomSearchScreen / ExploreRoomsScreen:
    • Filtros: Localização, Data e Hora, Tipo de sala, Capacidade, Comodidades.
    • Exibição dos resultados: Lista de salas disponíveis com foto, nome, capacidade, preço/hora.

Detalhes da Sala

  • RoomDetailUserScreen:
    • Carrossel de fotos, descrição completa, comodidades, capacidade, localização (mapa).
    • Preço, calendário de disponibilidade detalhada.
    • Seletores para data e horário de início e fim da reserva.
    • Botão "Reservar Agora".

Processo de Reserva

  • BookingSummaryScreen: Resumo da sala, data, horário, duração, preço total. Campos para informações adicionais. Botão "Prosseguir para Pagamento" / "Confirmar Reserva".
  • PaymentScreen: (Se integrado com gateway) Opções de pagamento, formulário de dados.
  • BookingConfirmationScreen: Mensagem de sucesso, Número de Protocolo, detalhes da reserva, instruções.

Minhas Reservas

  • MyBookingsScreen: Lista das reservas do usuário (Próximas, Passadas, Canceladas). Exibe sala, data/hora, status, protocolo.
  • BookingDetailUserScreen (View): Detalhes de uma reserva. Opção de "Cancelar Reserva", "Ver no Mapa".

Perfil do Usuário

  • UserProfileScreen: Visualizar e editar dados do perfil.
  • PaymentMethodsUserScreen: (Opcional) Gerenciar métodos de pagamento salvos.
  • NotificationSettingsUserScreen: Configurar preferências de notificação.
  • Help/SupportScreen: Informações de contato, FAQ.

Backend API: Autenticação (`/auth`)

Endpoints responsáveis pelo registro, login e gerenciamento de senhas dos usuários.

POST /signup (usuário final)
POST /login (ambos os perfis)
POST /admin/signup (criação de admins)
POST /forgot-password
POST /reset-password

Backend API: Recurso Usuários (`/users`)

Endpoints para gerenciamento de informações de perfis de usuários.

GET /me (perfil do usuário logado)
PUT /me (atualizar perfil)
GET / (Admin: listar usuários)
GET /:id (Admin: ver usuário específico)

Backend API: Recurso Espaços (`/spaces`)

Endpoints para administradores gerenciarem os espaços de coworking.

POST / (criar espaço)
GET / (listar espaços)
GET /:id (detalhes do espaço)
PUT /:id (atualizar espaço)
DELETE /:id (deletar espaço)

Backend API: Recurso Salas (`/rooms`)

Endpoints para gerenciamento de salas (criação, atualização, listagem para usuários e admins).

POST / (Admin: criar sala)
PUT /:id (Admin: atualizar sala)
DELETE /:id (Admin: deletar sala)
POST /:id/maintenance (Admin: adicionar período de manutenção)
DELETE /:id/maintenance/:maintenanceId (Admin: remover período de manutenção)
GET / (Público/Usuário: listar salas com filtros)
GET /:id (Público/Usuário: detalhes públicos da sala)

Backend API: Recurso Reservas (`/bookings`)

Endpoints para usuários criarem e gerenciarem suas reservas, e para administradores gerenciarem todas as reservas.

POST / (Usuário: criar nova reserva)
GET /my-bookings (Usuário: listar suas reservas)
GET /my-bookings/:id (Usuário: detalhes de sua reserva)
PUT /my-bookings/:id/cancel (Usuário: cancelar sua reserva)
GET / (Admin: listar todas as reservas)
GET /:id (Admin: detalhes de qualquer reserva)
PUT /:id/status (Admin: atualiza status da reserva)
POST /:id/payment (Admin: registra pagamento manual)

Considerações Adicionais

Além do modelo de dados, telas e API, o desenvolvimento do sistema de coworking deve levar em conta os seguintes aspectos cruciais para o sucesso e robustez da aplicação:

  • Pagamento: Integração com um gateway de pagamento (Stripe, PagSeguro, Mercado Pago, etc.) é fundamental. Definir se o pagamento é obrigatório no ato da reserva ou se pode ser feito depois/no local.
  • Notificações: Implementar notificações por email (usando, por exemplo, Nodemailer) e/ou push (via FCM - Firebase Cloud Messaging) para confirmações, lembretes, cancelamentos de reserva, alterações e solicitações de pagamento.
  • Disponibilidade em Tempo Real: A lógica para verificar a disponibilidade das salas precisa ser robusta e eficiente, consultando as coleções `bookings` e `rooms.maintenancePeriods` para o intervalo de tempo desejado, evitando conflitos de agendamento.
  • Segurança: Validar todas as entradas de dados, usar HTTPS para comunicação, proteger contra vulnerabilidades comuns (XSS, CSRF, SQL Injection - embora NoSQL, princípios de sanitização se aplicam), e armazenar senhas com hash seguro (ex: bcryptjs).
  • Testes: Planejar e executar testes unitários, de integração e End-to-End (E2E) para garantir a qualidade e confiabilidade do software.
  • Escalabilidade: Projetar a arquitetura pensando em crescimento futuro, tanto em número de usuários quanto em volume de dados e transações.
  • Usabilidade (UX/UI): Além das telas listadas, focar em uma interface intuitiva e uma experiência de usuário agradável em todo o aplicativo React Native.
